Contexte

Application de gestion de magasin.

Dans le cadre d un stage ou d une alternance, ce projet repond au besoin de centralisation des operations internes d un magasin : planning, conges, paie et taches.

Objectifs

Centralisation

Centraliser la gestion des employes et des plannings.

Suivi RH

Faciliter le suivi des conges et des absences.

Automatisation

Automatiser les regles metier liees au planning et aux contraintes de contrat.

Utilisateurs

  • Employe : consulte son planning, ses conges et ses fiches de paie.
  • Manager : organise les plannings et attribue les taches.
  • RH : valide les conges et gere les donnees administratives.
  • Administrateur : gere les acces et la configuration globale.

Fonctionnalites principales

  • Authentification securisee avec gestion des roles.
  • Gestion des plannings : creation, modification, consultation.
  • Gestion des conges : demande et validation.
  • Consultation des fiches de paie.
  • Gestion des taches avec code couleur.
  • Interface adaptee selon le role utilisateur.

Stack

Technologies utilisees.

React API REST Sequelize Docker
  • Front-end : React.
  • Back-end : API REST.
  • Base de donnees : via Sequelize.
  • Environnement : Docker.

Contraintes

  • Maximum 5 jours travailles par semaine.
  • Respect des contraintes liees aux contrats.
  • Gestion des acces selon les roles (RBAC).
  • Securisation des donnees sensibles.
